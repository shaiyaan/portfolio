Blue Bike Prediction

Machine learning model to predict and optimize Blue Bike availability across Boston

View Code

Problem Statement

Bluebike, a prominent bike-sharing service in Boston, faces challenges in understanding and optimizing trip durations for diverse user groups. Accurate trip time predictions are crucial for operational efficiency, improving customer satisfaction, and managing bike availability.

This code aims to solve these issues by predicting trip durations using historical data, identifying factors that influence trip times, and classifying trips based on their duration. This insight empowers Bluebike to make data-driven decisions about fleet management, pricing, and user experience enhancements.

My Approach

Data Preprocessing

  • •

    Calculated total trip times and distances using start and end geolocations.

  • •

    Added features like day of the week, membership type, bike type, and normalized Haversine distance.

  • •

    Filtered datasets to focus on specific user groups and trip scenarios (e.g., member-only trips, weekday trips).

Modeling

  • •

    Linear Regression Models: Predicted trip durations using location data and engineered Haversine distance. Improved accuracy by applying filters (e.g., removing long trips and casual users).

  • •

    Logistic Regression Models: Classified trips as short (<30 min) or long (≥30 min) using key features like normalized distance and membership type.

Evaluation and Visualization

  • •

    Used RMSE to measure the performance of regression models.

  • •

    Visualized model performance through residual and actual vs. predicted plots.

  • •

    Assessed logistic regression with accuracy and recall metrics to understand classification effectiveness.

Key Learnings

  • •

    Learned how incorporating Haversine distance calculations between start and end locations significantly improved the model's ability to predict trip durations, demonstrating the importance of spatial features in transportation models.

  • •

    Discovered that focusing on specific user segments and trip types (like member-only or weekday trips) led to more accurate predictions, highlighting how targeted analysis can reveal patterns that might be obscured in the full dataset.

  • •

    Developed a deeper understanding of model performance through multiple evaluation techniques, combining visual tools like residual plots with statistical metrics such as RMSE and recall to gain a complete picture of model effectiveness.
